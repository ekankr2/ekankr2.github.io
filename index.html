Phase 1: 생산성 이슈.

React + Spring Boot + MySQL 조합은 운영진의 잦은 요구사항 변경과 빠른 개발 요구에 적합하지 않았습니다. 신속한 의사결정에 대응하기 위해 Next.js, FastAPI, MongoDB 등 생산성이 높은 기술 스택으로 전환했습니다.

Phase 2: 서버 이슈.

FastAPI에서 응답 시간이 긴 외부 API 호출 시 블로킹 현상이 발생했습니다. 프로세스 수를 늘려도 개선되지 않아, 서버를 Vercel로 전환하여 서버리스 기반 병렬처리를 구성했습니다. PostgreSQL로 전환하여 안정성을 확보하고, Nginx 프록시 서버를 구축하여 서버리스 환경에서도 고정 IP를 확보했습니다.

서버리스 환경에서 PostgreSQL과의 커넥션 풀 문제가 지속적으로 발생했습니다. 동시에 ISMS 인증 요구사항에 따라 모든 서비스를 내부망에 구성해야 했습니다. 이를 해결하기 위해 AWS VPC 기반 전용 인프라로 전환하고 ECS를 도입하여 컨테이너 기반 배포 환경을 구축했습니다. 또한 Zone 분리 보안 원칙에 따라 기존 데이터베이스 RDS Aurora 로 이관하고, 별도 서브넷으로 분리하여 보안성을 강화했습니다. 이 과정에서 기존 Nextjs 코드 재사용을 위해 Node.js, Fastify를 도입했습니다. 결과적으로 ISMS 인증을 통과하였습니다.
